Expand description

Derive new session keys after a rekey with PCS protection.

This function provides Post-Compromise Security (PCS) by mixing in the rekey_auth_key which is derived from the static DH during handshake. An attacker who compromises session keys cannot derive future rekey keys without knowing the static DH secret.

Per 1-SECURITY.md (PCS fix):

ikm = ephemeral_dh || rekey_auth_key
(new_initiator_key, new_responder_key) = HKDF-Expand(
    ikm,
    "nomad v1 rekey" || LE32(epoch),
    64
)
